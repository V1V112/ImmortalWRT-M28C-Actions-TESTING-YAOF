# OpenWrt 私人仓库集成 - 版本 1.1.0 更新总结

**发布日期**: 2026-05-24  
**版本**: 1.1.0  
**主要改进**: /usr/bin 二进制文件智能处理

---

## 核心改进

### 问题解决

**问题**：使用私人配置仓库时，`/usr/bin` 目录下的压缩二进制文件无法被正确识别和处理，压缩包被直接复制到固件而不是解压。

**根本原因**：`fetch-custom-config.sh` 脚本使用简单的 `find + cp` 操作，没有实现 `stage-overlay.sh` 中的智能识别逻辑。

**解决方案**：将 `stage-overlay.sh` 中关于 `/usr/bin` 处理的核心函数移植到 `fetch-custom-config.sh`，实现完整的智能处理流程。

---

## 技术实现

### 代码修改

#### 1. scripts/fetch-custom-config.sh 增强

**添加的核心函数**（共 5 个）：

```bash
1. strip_archive_ext()         # 去除压缩包扩展名
   - 支持 11 种常见格式
   - 用于生成首选文件名

2. is_archive()                # 检测压缩包
   - 基于文件扩展名
   - 大小写不敏感

3. extract_archive()           # 解压压缩包
   - 支持 9 种解压方式
   - 自动创建临时目录
   - 包含错误处理

4. candidate_score()           # 评分候选二进制
   - 基于文件类型（ELF/脚本）
   - 基于目标架构匹配度
   - 基于文件属性（权限、名称等）
   - 最高 170+ 分，最低 -999 分

5. install_detected_binary()   # 选择安装最佳二进制
   - 递归扫描目录
   - 自动选择最高分二进制
   - 设置执行权限 (755)
```

**处理流程改进**：

```bash
原流程（1.0.0）:
├─ 使用 find 查找所有文件
└─ 逐个复制（忽略 /usr/bin）

新流程（1.1.0）:
├─ 排除 /usr/bin 的其他文件处理
└─ 独立处理 /usr/bin：
   ├─ 对每个项目（文件/目录/压缩包）
   ├─ 如果是压缩包 → 解压 → 扫描 → 评分 → 安装
   ├─ 如果是目录 → 扫描 → 评分 → 安装
   └─ 如果是文件 → 直接复制
```

**关键修复**：

- ✅ 使用 `< <()` 替代管道避免子 shell 变量丢失
- ✅ 移除不必要的 `local` 声明（顶级脚本作用域）
- ✅ 实现本地优先级保护（检查文件存在后跳过）

## 功能特性

### 支持的压缩格式

| 格式 | 命令 | 用途 |
|------|------|------|
| .tar.gz / .tgz | `tar -xzf` | 最常见，高压缩率 |
| .tar.xz / .txz | `tar -xJf` | 最高压缩率 |
| .tar.bz2 / .tbz2 | `tar -xjf` | 中等压缩率 |
| .tar.zst | `tar --zstd` | 现代高效格式 |
| .tar | `tar -xf` | 无压缩 tar 包 |
| .zip | `unzip` | 跨平台格式 |
| .gz | `gzip -dc` | 单文件压缩 |
| .xz | `xz -dc` | 单文件高压缩 |
| .zst | `zstd -dc` | 单文件现代格式 |

### 评分系统详解

**评分维度**：

1. **文件类型**（最高权重）
   - ELF 二进制: +100 分（最优）
   - 脚本 (shebang): +65 分
   - 脚本 (文件识别): +55 分
   - 其他: -999 分（排除）

2. **目标架构匹配**（关键）
   - aarch64/arm64: +45 分（for ELF）或 +25 分（for path）
   - arm: +20 分
   - x86/amd64/Darwin: -80 分（不兼容）

3. **文件特征**（微调）
   - 文件名匹配提示: +35 分
   - 无扩展名: +8 分
   - 路径含 linux: +10 分
   - 执行权限: +5 分

4. **自动排除的文件类型**
   - README*, LICENSE*, COPYING*
   - *.md, *.txt, *.json, *.conf
   - *.service, *.desktop
   - *.png, *.jpg, *.jpeg, *.gif, *.svg

**例子**：

```
文件: app-aarch64 (ELF)
计算: 100 (ELF) + 45 (aarch64) + 25 (aarch64 in path) = 170 ✅

文件: app-arm (ELF)
计算: 100 (ELF) + 20 (arm) = 120 ✅

文件: README.md
计算: -999 (文档) ❌ 排除

结论: app-aarch64 被选中
```

### 优先级系统

三层优先级（从高到低）：

```
本地 files/usr/bin/app         ← 最高，不被覆盖
    └─ 私人仓库 /usr/bin/app.tar.gz  ← 如本地已存在则跳过
        └─ 解压内容选择         ← 自动选择最优版本
```

## 使用指南

### 配置私人仓库

**推荐结构**：

```
your-private-repo/
├── files/
│   ├── etc/
│   │   └── config/
│   │       └── network
│   └── usr/
│       └── bin/
│           ├── app1.tar.gz        # 压缩格式
│           ├── app2               # 直接二进制
│           └── tools/             # 目录格式
│               ├── tool-aarch64
│               └── tool-arm
├── README.md
└── .gitignore
```

### 快速开始

**1. 创建压缩包二进制**：

```bash
# 创建包含多个架构的压缩包
app.tar.gz
├── app-aarch64           ← 自动选中
├── app-arm
└── app-x86_64            ← 跳过（不兼容）
```

**2. 运行构建**：

```bash
export CUSTOM_CONFIG_REPO_URL="https://github.com/you/configs.git"
export CUSTOM_CONFIG_BRANCH="main"
export CUSTOM_CONFIG_TOKEN="your_token"

# 脚本自动处理 /usr/bin 二进制
```

**3. 验证结果**：

```bash
# 检查安装的二进制
ls -la openwrt/files/usr/bin/
file openwrt/files/usr/bin/app1
```

## 故障排查快速查询

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 压缩包未解压 | 不支持格式或无效内容 | 验证格式和内容 |
| 选错架构版本 | 评分系统匹配度低 | 规范文件命名 |
| 脚本无法执行 | 缺少 shebang 或权限 | 检查文件格式 |
| 本地文件被覆盖 | 优先级逻辑错误 | 检查脚本版本 |
| 编译失败 | 二进制不兼容 | 验证文件格式 |

## 已知限制

1. ⚠️ **符号链接**：压缩包内的符号链接会被复制为普通文件
2. ⚠️ **权限**：所有二进制统一设置为 755 权限
3. ⚠️ **架构**：仅支持自动选择，不支持安装多个架构版本
4. ⚠️ **单一仓库**：当前只支持一个私人仓库
